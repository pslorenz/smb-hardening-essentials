# SMB Hardening Essentials (Tier 1)

**Opinionated, low-friction security baseline for small and mid-sized businesses without in-house IT.**

Runs two open-source assessors (Maester for Microsoft 365/Entra, HardeningKitty for Windows endpoints) against a curated Tier 1 control set, then produces a single HTML report mapped to CIS IG1 that a non-technical business owner can actually read.

> **Scope:** 25-200 seats, no dedicated IT, cost-sensitive, no active compliance mandate.
> For 150+ seats, in-house IT, or compliance obligations (HIPAA, SOC 2, CMMC, PCI), see **smb-hardening-enhanced** (Tier 2).

---

## Why this exists

CIS Benchmarks, Microsoft Security Baselines, and the Defender defaults are all fine. They produce reports with 300+ findings. That's useful for a Fortune 500 with a SOC. For a 40-person accounting firm without a sysadmin, it's paralysis.

This baseline is the ~100 controls across M365 and Windows that:

1. Close a real attack path I've see exploited in the field (not a theoretical one).
2. Deploy without generating help-desk tickets in an org without help-desk capacity.
3. Produce a finding a business owner understands "MFA is off for your admin account" not "CIS 1.1.1 non-compliant."

Everything excluded from this baseline is documented with a reason. See [docs/BASELINE.md](docs/BASELINE.md).

## Quickstart

```powershell
# One-time: install prerequisites
Install-Module -Name Maester -Scope CurrentUser
Install-Module -Name Microsoft.Graph.Authentication -Scope CurrentUser

# HardeningKitty: install as a module. Either:
#   (a) Copy HardeningKitty.psd1, HardeningKitty.psm1, and lists/ from
#       https://github.com/scipag/HardeningKitty into a PSModulePath location, OR
#   (b) Clone the repo anywhere and pass -HardeningKittyPath to the assessment script.
# HardeningKitty is not published to PowerShell Gallery as of this writing; install from the GitHub release.

# Assess a tenant + local endpoint
.\Invoke-SmbHardeningAssessment.ps1 -TenantId <guid> -ClientName "Acme Corp" -OutputPath .\reports

# Tenant only (no endpoint check — useful for initial sales calls)
.\Invoke-SmbHardeningAssessment.ps1 -TenantId <guid> -ClientName "Acme Corp" -CloudOnly

# Endpoint only (useful for imaged-machine verification)
.\Invoke-SmbHardeningAssessment.ps1 -ClientName "Acme Corp" -EndpointOnly
```

Output lands in `reports/<ClientName>/<YYYYMMDD-HHMMSS>/`:

- `report.html` the client-facing report
- `maester-raw.json` full Maester output
- `hardeningkitty-raw.csv` full HardeningKitty output
- `findings.json` normalized, mapped to CIS IG1 control IDs
- `summary.txt` one-page executive summary

A rendered example lives in [examples/sample-report.html](examples/sample-report.html).
Yes, the HTML report was generated by AI. I have zero design/make this pretty skills.

## What this is not

- **Not a replacement for Defender for Business** or any licensed security product. This measures configuration, not behavior.
- **Not continuous monitoring.** For that, schedule the wrapper via Task Scheduler or your RMM and diff the `findings.json` between runs. Drift monitoring is a Tier 2 feature.
- **Not remediation.** The report tells you what's wrong and links to Microsoft docs. Fixing it is a separate engagement.
- **Not a compliance attestation.** Mapping to CIS IG1 is directional, not audit-grade. For audit-grade evidence, you want Tier 2's compliance-framework mapping.
- **Not for enterprises.** If you have a SOC, a dedicated security team, or more than ~500 seats, the Tier 2 baseline or full CIS IG2/IG3 tooling is the right fit.

## Prerequisites

**On the machine running the assessment:**

- Windows 11 or Windows Server 2019+
- PowerShell 7.2+
- Maester (`Install-Module Maester`)
- HardeningKitty a PowerShell module (`.psd1` + `.psm1`) distributed via [scipag/HardeningKitty](https://github.com/scipag/HardeningKitty) on GitHub. Install into a PSModulePath location, or clone and pass `-HardeningKittyPath` to the assessment script.
- Microsoft Graph PowerShell SDK (`Install-Module Microsoft.Graph.Authentication`)

**In the M365 tenant:**

- An account with at least Global Reader and Security Reader.
- For a full assessment, also SharePoint Administrator and Exchange Administrator (read-only access to some settings still requires these role memberships).
- For unattended runs, an app registration with `Policy.Read.All`, `Directory.Read.All`, `Organization.Read.All`, `Reports.Read.All`, `AuditLog.Read.All`, `SharePointTenantSettings.Read.All`, and certificate-based auth (client secrets are not supported for SharePoint).

## Customization

**Running against only some categories:**

```powershell
.\Invoke-SmbHardeningAssessment.ps1 -TenantId <guid> -ClientName "Acme" `
    -Categories Identity,ExchangeOnline
```

**Excluding specific controls** (for a client where a control legitimately doesn't apply):

Create a `client-exceptions.json` in the output path before running:

```json
{
  "exclusions": [
    {
      "controlId": "SMB-NET-004",
      "reason": "Legacy Epson label printer requires NetBIOS; compensating control: VLAN isolation",
      "approvedBy": "client IT owner",
      "expiresOn": "2026-12-31"
    }
  ]
}
```

Exclusions appear in the report as acknowledged risk acceptance, not as passes.

## Contributing

Field feedback is the whole point. If a control breaks something in your environment, please open an issue with:

- Control ID and what it does
- What broke (be specific: app name, version, symptoms)
- Your environment profile (seats, M365 SKU, on-prem AD yes/no, LOB apps)

PRs adding exclusion annotations, improving remediation links, or expanding the runbook are welcome. PRs adding controls will be evaluated against the Tier 1 test ("closes a real attack path, low friction, business-owner-explainable"). Controls that fail that test belong in Tier 2.
